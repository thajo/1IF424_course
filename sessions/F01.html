Webbens utveckling



Statiska webbsidor (Server/Klient)

  • Du skriver informationen i HTML lägger dem på webbservern
  • Ska informationen ändras skriver du om HTML-koden i sidorna.

Men hur funkar merparten av webbplatserna?

  • Vem skriver all HTML, CSS och javascript för att visa denna lista?
  • Finns det en sida för VARJE tänkbart sökord?
  • Vart sparas informationen?

Något saknas!

Kod som kör i webbservern kan skapa HTML-kod beroende på vad webbläsaren skickar in

Dynamisk webbapplikation

Dynamisk webbapplikation. Du skriver kodfiler (PHP) som läggs på servern Server exekverar dessa och HTML skapas på olika sätt beroende på vilka HTTP

Beslut som tas på servern (backend)

  • Vad vill användaren göra? Hämta? Posta? Vilken data skickas med?
  • Vem skickar detta? Har personen rätt att posta?
  • Vart ska informationen sparas? Hur? Är det korrekt data? (validering)
  • Skicka tillbaka ett begripligt svar till webbläsaren (HTML)

Varför utvecklar vi webbapplikationer?

Vad är fördelarna mot så kallade skrivbordsapplikationer (desktop applications) som t.ex. Word, Photoshop o.s.v.

  • Inga installationer (mer än en webbläsare)
  • Plattformsoberoende, platsoberoende
  • Alltid senaste versionen, enklare distrubation
  • Tänk om facebook varit en skrivbordsapplikation?

Alla typer av applikationer passar inte som webbapplikation. Krav på prestanda, säkerhet o.s.v.

Jo, men vad ska vi GÖRA?

  • Kursens första del handlar om PHP-programmering.
  • PHP är ett serverscript, det exekveras på en webbserver
  • Ni ska alltså få förståelse och kunskap kring att skapa dynamiska webbapplikationer
  • Wordpress är skrivet i PHP, med kunskap i PHP kan man "enkelt" skapa kompletta webbplatser

Ytterligare orsaker till att kunna PHP?

  • Förståelse för utvecklingsprocessen - programmering
  • Vanligt serverscriptspråk idagsläget - stort stöd på webbhotell
  • Wikipedia, facebook, 65% av alla webbplatser?
  • Det finns flera serverscriptspråk, C# (ASP.NET), Ruby, Python o.s.v.
